# Tape Sorting

Данный проект реализует алгоритм внешней сортировки для устройства хранения типа лента.  
Считываются целые числа (int32_t) из входной ленты, сортируются с использованием ограниченного объёма оперативной
памяти и записываются в выходную ленту.  
Операции с лентой (чтение/запись/сдвиг/перемотка) могут быть сконфигурированы с искусственными задержками через текстовый
конфигурационный файл.

## Возможности

- Интерфейс `ITape` с настраиваемыми задержками.
- Реализация `FileTape`, которая симулирует ленту через обычный бинарный файл.
- Внешняя сортировка с ограниченным использованием памяти.
- Загрузчик конфигурации из файлов формата `ключ=значение`.
- Система сборки CMake с пресетами и поддержкой тестов.
- Юнит тесты с использованием gtest.

## Требования

- Компилятор с поддержкой C++20.
- CMake 3.28 или выше.
- Ninja (рекомендуется для использования пресетов) или любой другой генератор CMake.
- gtest для тестов.

## Сборка

Проект использует пресеты CMake. Можно собрать приложение либо тесты.

Сборка приложения (Release):

    cmake --preset Release

    cmake --build --preset Release

Исполняемый файл tape появится в build/Release/app.

Сборка тестов (Debug):

    cmake --preset Test-Debug

    cmake --build --preset Test-Debug

Исполняемый файл tape_tests появится в build/Test-Debug/tests.

Все доступные пресеты:

Release, Debug, RelWithDebInfo, Test-Release, Test-Debug, Test-RelWithDebInfo.

## Запуск приложения

    ./build/Release/app/tape <входной файл> <выходной файл> <конфиг>

`входной файл` - путь к входной ленте (бинарный файл с 32-битными целыми числами).
`выходной файл` - путь для записи отсортированной ленты.
`конфиг` - путь к конфигурационному файлу.

## Конфигурационный файл

Конфигурационный файл - обычный текстовый файл с парами `ключ=значение`.

Все ключи обязательны. Пример:

    read_delay_ms=10
    write_delay_ms=20
    move_delay_ms=5
    rewind_delay_ms=30
    memory_limit_bytes=1024
